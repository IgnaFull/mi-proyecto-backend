Tu aplicación frontend alojada en Vercel necesita conectarse a tu backend mediante peticiones HTTP.

// Ejemplo de función para obtener comentarios desde el backend

    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Error fetching comments:', error);

Backend (Express.js)

Tu backend en Express.js debe tener endpoints para interactuar con la base de datos MongoDB.

// Ejemplo de endpoint en Express.js
const express = require('express');
const mongoose = require('mongoose');
const app = express();

// Conectar a MongoDB
mongoose.connect(process.env.MONGODB_URI, {
  useNewUrlParser: true,
  useUnifiedTopology: true
});

// Ruta para obtener comentarios
app.get('/api/comments', async (req, res) => {
  try {
    const comments = await Comment.find();
    res.json(comments);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

// Iniciar servidor
const PORT = process.env.PORT || 5000;
app.listen(PORT, () => {
  console.log(`Servidor ejecutándose en puerto ${PORT}`);
});

Base de Datos (MongoDB)

Tu base de datos MongoDB Atlas almacena los datos y se conecta con tu backend mediante el driver de MongoDB.

// Ejemplo de modelo de Mongoose para comentarios
const mongoose = require('mongoose');

const commentSchema = new mongoose.Schema({
  name: {
    type: String,
    required: true
  },
  email: {
    type: String,
    required: true
  },
  text: {
    type: String,
    required: true
  },
  date: {
    type: Date,
    default: Date.now
  }
});

module.exports = mongoose.model('Comment', commentSchema);

Configuración de Despliegue

Frontend: Ya tienes tu frontend desplegado en Vercel.
Backend: Debes desplegar tu backend en Vercel, Netlify, Heroku o Railway.

Pasos para desplegar el backend:

  1. Crea un archivo vercel.json o configura el servidor para producción
  2. Asegúrate de que tu package.json tenga un script "start": "node server.js"
  3. Configura las variables de entorno en la plataforma de despliegue
  4. Conecta tu repositorio de GitHub para despliegues automáticos
// Ejemplo de vercel.json para backend en Vercel
{
  "version": 2,
  "builds": [
    {
      "src": "server.js",
      "use": "@vercel/node"
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "server.js"
    }
  ]
}
